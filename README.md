## 三工段流程

```
工段1：角色九宫格定妆（即梦图生图）
         ↓
工段2：动作首帧图生成（GPT-img2 / 香蕉）
         ↓
工段3：动作视频提示词（视频提示词智能体）
         ↓（用户要求时）
工段3续：Seedance2-skill 生成视频
```

| 工段 | 输入 | 工具 | 输出 |
|------|------|------|------|
| 工段1 | 角色原型参考图 | 即梦图生图 | 九宫格定妆照 |
| 工段2 | 姿态参考图 + 九宫格定妆照 | GPT-img2（首选）/ 香蕉（次选） | 动作首帧图 |
| 工段3 | 动作首帧图 + 动作描述 | 视频提示词智能体 | 视频提示词文本 |
| 工段3续 | 视频提示词文本 | Seedance2-skill | 视频文件 |

## 工段1：九宫格定妆

### 触发词

`游戏关键帧定妆` / `游戏角色定妆` / `九宫格定妆` / `角色定妆照` / `定妆提示词`

### 输入

用户上传 1 张角色原型参考图

### 输出

生成 3×3（9个视角）的网格布局提示词 → 调用即梦图生图

### 九宫格视角

| 序号 | 视角 |
|------|------|
| 1 | 正面头像：坐姿脸部特写 |
| 2 | 左侧头像：左侧45度脸部特写 |
| 3 | 右侧头像：右侧45度脸部特写 |
| 4 | 正面行走头像：正面视角略微俯视，行走姿态 |
| 5 | 正面全身坐姿：坐姿全身像 |
| 6 | 背面全身坐姿：背面坐姿全身像 |
| 7 | 背面行走后视图：纯背面，无正面特征 |
| 8 | 正侧面行走全身：四肢一前一后 |
| 9 | 侧45度坐姿全身：前腿并拢，后腿蜷缩 |

### 调用方式

```
即梦 image2image：
- 参考图：上传角色原型参考图
- 提示词：使用生成的九宫格定妆提示词
- 比例：1:1 / 分辨率：2K
```

## 工段2：动作首帧图

### 触发词

`首帧图` / `首帧生成` / `生成首帧` / `动作首帧图`

### 输入

- **图1**：姿态参考图（构图来源，用户从网络获取或自行设计）
- **图2**：工段1输出的九宫格定妆照（角色特征来源）

### 核心提示词

```
【图2】是一张角色九宫格定妆图，请用【图2】的角色，替换【图1】的角色，请分析【图2】提取出准确的角色特征，确保角色一致性；最终按照【图1】的构图生成图片，画幅比例1:1
```

### 工具优先级

1. **GPT-img2**（首选）- AIX 平台
2. **香蕉（nano）**（次选）

## 工段3：视频提示词

### 触发词

`动作视频提示词` / `视频提示词` / `生成视频提示词`

### 输入

- **首帧图**：工段2输出的动作首帧图（必须使用）
- **动作**：用户指定（如：正面行走、待机、洗澡、喂食等）
- **视角**：用户指定（正面/侧面/背面）
- **备注**：用户可选，补充动作细节/起始姿态

### 全局硬约束

| 约束项 | 要求 |
|--------|------|
| 时长 | 固定4秒 |
| 循环 | 动作必须无缝循环，首尾状态自然衔接 |
| 镜头 | 全景固定镜头 |
| 运镜 | 无运镜（禁止推拉摇移旋转） |
| 角色位置 | 全身出镜，始终在画面中央不位移 |
| 空间 | 禁止空间透视变化（禁止变大/变小） |
| 道具 | 画面中不出现任何道具/手/食物碗等外物 |
| 背景 | 纯白背景（干净无纹理/无阴影/无噪点） |
| 画幅 | 1:1 |

### 已验证动作映射

| 动作 | 描述要点 |
|------|---------|
| 正面行走 | 四肢交替踏步清晰，节奏稳定，表情自然 |
| 背面行走 | 四肢踏步清晰，身体轻微上下起伏，尾巴自然左右摆动 |
| 侧面行走 | 四肢交替踏步清晰，身体轻微上下起伏，尾巴轻微摆动，表情自然 |
| 待机 | 轻微呼吸起伏，尾巴慢摆，耳朵轻抖 |
| 闭眼 | 睁眼→闭眼清晰可辨，循环恢复起始状态 |
| 洗澡 | 身体轻扭/抖动，表情开心享受，眯眼微笑，耳朵轻抖，尾巴摆动（无道具） |
| 喂食 | 头部轻快啄食节奏，身体微微起伏，眼睛亮晶晶，嘴角上扬做舔嘴回味状，尾巴兴奋摆动（无道具） |
| 被抚摸 | 眯眼放松，嘴角微扬，身体蹭靠配合，尾巴慢摆，表情舒服享受（无道具） |
| 后脚挠头 | 头部歪向一侧，身体微微侧倾，后腿抬起挠痒，表情舒服，尾巴轻摆（无道具） |

### 输出格式

纯文本（不用代码块），中英双语，可直接复制使用：

```
【中文】
[动作视角]动画，纯白背景，角色全身居中，动作流畅连贯，首尾无缝衔接，像素游戏风格。

【English】
Reference image 1 as the character reference. Generate a [视角]-facing [角色类型] [动作类型] animation in place. Clean white background. Character centered in frame. Smooth motion, seamless loop. Pixel art style.
```

> ⚠️ 角色外观特征由参考图锁定，提示词中不再描述角色外观。肢体结构约束和配件动态描述已精简。

## 工段3续：视频生成（可选）

### 触发词

`生成视频` / `视频生成`

### Seedance2-skill 依赖说明

安装方式（git clone）：
```bash
git clone https://github.com/leonluo2008-ops/Seedance2-skill.git
```

环境要求：
- Python 3.6+
- ARK_API_KEY（需在火山引擎控制台申请）

如在其他平台使用，需替换为等价的视频生成 API 调用。

### 执行方式

调用 Seedance2-skill 生成视频：

```bash
python3 scripts/seedance.py create \
  --image <首帧图路径> \
  --prompt "<工段3输出的提示词文本>" \
  --ratio adaptive \
  --duration 4 \
  --generate-audio false \
  --wait
```

## 历史验证案例

### 太空宇航橘猫（像素风）

**工段1**：即梦图生图，3×3九宫格视角，像素游戏风格（Pixel Art），黑色1像素粗描边，赛璐璐着色。

**工段2**：GPT-img2（首选），角色九宫格定妆照 + 姿态参考图 → 首帧图，验证通过 ✅

**工段3**：
- 洗澡动画 ✅（cgt-20260424185359-wzqgc）
- 吃东西动画 ✅（cgt-20260424215247-h98f9）
- 洗澡后甩水动画 ✅（cgt-20260424220710-nx9gn）

**核心经验**：触发"被 X"类动作会生成 X 道具（刷子/食物/抚摸的手）。正确做法是从角色反应出发描述情绪和身体反馈，不描述动作本身。
