# 工段3参考：动作视频提示词规范

> 更新：2026-04-25（基于 Seedance2-skill 验证标准 + 实测教训）
> 本文件内容已与 Seedance2-skill 的 prompt 结构完全对齐

---

## 身份与目标

专门为 Seedance 2.0（doubao-seedance-2-0-fast-260128）编写稳定的像素风动作视频提示词。
目标：在严格保持角色一致性与镜头稳定性的前提下，把"动作名称/视角/细节要求"转写成可直接复制使用的单条视频提示词。

---

## 执行前审核流程（必须遵守）

> **教训：视频生成成本高，必须先确认再执行**

1. **识别参考图** → 描述图片内容（角色、视角、姿态、背景）
2. **与用户确认** → 等待用户明确回复"确认"后才执行
3. **不确定的地方** → 必须先问清楚，不能猜测执行

**反面案例**：用错参考图（如用原型图代替用户提供的新参考图），导致生成结果不符合预期，浪费 API 费用。

---

## 输入（用户提供）

- **首帧图**：必须使用用户提供的参考图（不能随意替换）
- **动作**：行走/待机/伸懒腰/洗澡/喂食/后脚挠头/被抚摸/眯眼/打哈欠/尾巴摇摆/舔毛/蹭脸 等
- **视角**：正面/侧面/背面，**高角度俯视/平视/仰视**（视角必须在首帧图中体现）
- **备注**：用户可选，补充动作细节/起始姿态

---

## 全局硬约束（任何情况不可违背）

| 约束项 | 要求 |
|--------|------|
| 镜头 | 全景固定镜头 |
| 运镜 | 无运镜（禁止推拉摇移旋转） |
| 角色位置 | 全身出镜，始终在画面中央不位移 |
| 空间 | 禁止空间透视变化（禁止变大/变小） |
| 道具 | 画面中不出现任何道具/手/食物碗等外物 |
| 背景 | 纯白背景（干净无纹理/无阴影/无噪点） |
| 循环 | 动作必须无缝循环，首尾状态自然衔接 |
| 时长 | 固定4秒 |
| 风格 | 像素视频风格，严格按首帧图角色特征 |

---

## Prompt 标准结构（Seedance2-skill 验证格式）

**核心原则**：连贯段落式，不用结构化标签填充。

```
纯像素画风格，像素块大小统一，边缘锐利，无抗锯齿。全景固定镜头，【视角】平视，角色全身居中【起始姿态】。角色【动作反应描述】（见下方动作反应库），表情【表情描述】，动作流畅连贯，首尾无缝衔接。纯白背景，无任何道具或异物出现，禁止出现【禁止道具】（见禁止清单）。保持在画面中央不位移，无透视变化。
```

## 禁止清单（触发道具的问题）

> **教训：描述"被 X"类动作会触发 X 道具生成**

| ❌ 错误描述 | ✅ 正确替代 |
|-----------|-----------|
| 被刷洗/洗澡/清洁 | 抖毛甩动/毛发蓬松/眯眼舒服 |
| 吃东西/咀嚼 | 啄食节奏/舔嘴回味/鼻子抽动 |
| 被抚摸/挠痒 | 眯眼/蹭靠/尾巴慢摆 |
| 后脚挠头/挠痒 | 后腿抬起做出挠痒动作 |
| 水/水滴/水珠 | 抖毛甩动/毛发蓬松 |
| 两足直立行走 | 四足着地行走 |

## 视角注意事项

> **教训：视角由首帧图决定，无法在 prompt 中强制调整**

- **高角度俯视**：必须在首帧图阶段体现，不能靠 prompt 指令在视频生成时修改
- 首帧图视角不合适 → 应该在**工段2（首帧图生成）**阶段修复
- 工段3视频生成阶段硬调视角的成功率低，且浪费 API 费用

---

## Q 版角色行走问题

> **教训：Q 版比例角色容易被 Seedance 误解为直立人形**

- 太空猫等 Q 版角色（头大身小）容易被模型解读为两足直立人形
- 四足行走姿态必须在**首帧图中明确展示四足几何特征**
- 即使在 prompt 中强调"四足着地""禁止直立"，模型仍可能生成错误步态
- 解决方案：工段2首帧图必须使用能清晰展示四足行走姿态的参考图

- 纯文本（不用代码块）
- 放在消息正文中发送
